========================================
FAQ自動生成システム
========================================

【概要】
問い合わせデータから高品質なFAQを自動生成するシステムです。
問い合わせ履歴データをクレンジングし、重複を除去して最終的なFAQを作成します。

【必要な環境】
- Windows 10/11
- Python 3.9以上
- Azure OpenAI API（GPT-4.1 と Embedding API）
- インターネット接続

【初回セットアップ（5分）】

1. Python のインストール
   https://www.python.org/downloads/
   ※ インストール時に「Add Python to PATH」にチェック

2. ファイルの解凍
   このZIPファイルを適当なフォルダ(※「★」等の特殊文字を含まないフォルダ)に解凍

3. Azure OpenAI API の設定　※現行設定済なので、要変更の場合のみ実施

   a) .env をメモ帳で開く
   b) 以下の4つを入力:
      - AZURE_OPENAI_API_KEY
      - AZURE_OPENAI_ENDPOINT
      - AZURE_OPENAI_API_KEY_EMBEDDING
      - AZURE_OPENAI_ENDPOINT_EMBEDDING
   c) ファイルを保存

4. デフォルトFAQの配置（オプション）
   既存のFAQファイル（CSV）がある場合:
   data/default_faq/FAQ_list.csv として配置

【起動方法】

「start.bat」をダブルクリック

※ 初回起動時は自動でパッケージをインストールします（数分かかります）

【使い方】

1. ブラウザが自動で開きます
   （開かない場合: http://localhost:8501 にアクセス）

2. サイドバーからファイルをアップロード
   - 問い合わせ履歴CSV（必須）
     ※ 内部で既存パイプライン接続用のLegacy列名へ変換されます
   - 既存FAQ（デフォルトFAQがない場合は必須）

3. パラメータを調整（任意）
   - Phase 2（文字列類似度）閾値: 0.75（推奨）
   - Phase 3-1（Q内Embedding）閾値: 0.75（推奨）
   - Phase 3-2（既存FAQ）閾値: 0.75（推奨）

4. 「🚀 処理開始」ボタンをクリック

5. 処理完了後、結果をダウンロード


パラメータ設定
--------------------------------------------------------------------------------
パラメータ               | デフォルト | 説明
--------------------------------------------------------------------------------
Phase 2 閾値            | 0.75      | 文字列類似度がこの値以上で重複統合
Phase 3-1 閾値          | 0.75      | Q内Embedding類似度がこの値以上で重複統合
Phase 3-2 閾値          | 0.75      | 既存FAQとの類似度がこの値以上で削除
AI並列数                | 50        | Phase 1のAPI同時リクエスト数（10-100）
Embeddingバッチサイズ   | 100       | 一度にEmbedding処理する件数
--------------------------------------------------------------------------------

※ Phase 0の類似度閾値は0.90で固定（変更不可）

【トラブルシューティング】

Q: アプリが起動しない
A: 以下を確認してください
   - Pythonがインストールされているか
   - .envファイルが存在するか
   - start.bat を右クリック → 「管理者として実行」

Q: 環境変数エラーが出る
A: .envファイルの内容を確認してください
   - 4つの環境変数がすべて設定されているか
   - APIキーとエンドポイントが正しいか

Q: CSVファイルのエラーが出る
A: ファイルの内容を確認してください
   - 問い合わせデータ: 問い合わせ内容、回答内容、問い合わせ分類、キーワード
   - FAQ: 質問、回答
   - 文字コード: UTF-8

Q: 処理が終わらない
A: データ量によって時間がかかります
   - 100件: 約2-3分
   - 500件: 約5-8分
   - 1000件: 約10-15分


Q: API Rate Limit (429)のエラーが出る
A: 自動リトライ機能があります（最大5回、指数バックオフ）
   頻発する場合はAI並列数を下げてください（50→30等）


Q: エンコーディングエラーが出る
A: CSV保存時にUTF-8またはShift-JISで保存し直す
   Excelから保存する場合は「CSV UTF-8」を選択

Q: メモリ不足になる
A: 大量データ（3000件以上）の場合は分割処理を検討
   Embeddingバッチサイズを小さくする


【技術詳細】
Phase 0: 問い合わせ内容の重複チェック
クレンジング前の「問い合わせ内容」列で重複を検出し、
類似グループの代表データのみをAI処理に送ることでAPI節約を実現。

- 完全一致チェック: MD5ハッシュで高速検出
- 類似度チェック: TF-IDF + コサイン類似度 + 階層的クラスタリング
- 閾値: 0.90固定
- 代表選出: グループ内で回答内容の文字数が最も長いデータ
- 短文チェック: 回答内容が30文字以下のデータは削除


Phase 1: AIクレンジング
Azure OpenAI GPT-4.1を使用して問い合わせデータをFAQ形式に変換。

- FAQ適性判定: 作業代行依頼・マスタ登録依頼などを自動除外
- 質問生成: 固有名詞を保持しつつ、検索しやすい質問文に変換
- 回答生成: 回答内容に記載された情報のみを使用（推測禁止）
- 並列処理: 非同期処理で最大50並列（調整可能）
- リトライ: Rate Limit (429) エラー時は自動リトライ（最大5回）


Phase 2: 質問の重複チェック
クレンジング後の「質問」カラムで重複を検出。
Phase 0と同様のアルゴリズムだが、閾値は調整可能（デフォルト0.75）。


Phase 3: Embedding重複除去
■ Phase 3-1（Q内重複除去）
  Azure OpenAI text-embedding-3-largeを使用して質問の意味的な重複を検出。
  文字列が異なっていても意味が同じ質問を統合。

■ Phase 3-2（既存FAQ照合）
  既存FAQとの近さを確認し、完全一致・類似・更新候補・矛盾可能性としてレビュー対象化。


【推定APIコスト】
コストは以下の式で概算されます:

  推定コスト（円）= (代表件数 × 0.003 + FAQ候補件数 × 2 × 0.0001) × 150

例: 800件の問い合わせ、代表200件、FAQ候補150件の場合
  = (200 × 0.003 + 150 × 2 × 0.0001) × 150
  = (0.6 + 0.03) × 150
  = 約94.5円

※ Phase 0で類似データをグループ化することで、
   全件AI処理する場合と比較して大幅なコスト削減が可能
